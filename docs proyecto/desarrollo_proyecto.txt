# Desarrollo del Proyecto VendeYa

## 1. Definición del Proyecto
### 1.1 Descripción General del Proyecto
VendeYa es una plataforma de marketplace online que permite a los usuarios registrarse como compradores o vendedores, publicar productos, gestionar pedidos y realizar compras.

### 1.2 Tipos de Empresas. Sectores Productivos
El proyecto está orientado a empresas de comercio electrónico y sectores de venta al por menor.

## 2. Diseño y Fases del Proyecto
### 2.1 Objetivos. Especificación de Requisitos
- **Objetivo General:** Crear una plataforma de marketplace eficiente y fácil de usar.
- **Objetivos Específicos:**
  - Facilitar la compra y venta de productos.
  - Proveer herramientas de gestión para vendedores.
  - Implementar un sistema de mensajería entre usuarios.

### 2.2 Identificación de las Fases del Proyecto
- **Fase de Análisis:** Recolección de requisitos y análisis de necesidades.
- **Fase de Diseño:** Diseño de la arquitectura del sistema y la interfaz de usuario.
- **Fase de Implementación:** Desarrollo del frontend y backend.
- **Fase de Pruebas:** Pruebas unitarias e integrales para asegurar la calidad.

### 2.3 Herramientas de Diseño
- **Software Utilizado:** React para el frontend, PHP y MySQL para el backend.
- **Metodologías:** Desarrollo ágil con iteraciones cortas.

### 2.4 Evaluación Económica y Financiación del Proyecto
- **Estudio de Viabilidad:** Evaluación de costos y beneficios.
- **Conclusión de Viabilidad:** El proyecto es viable económicamente.
- **Ayudas Económicas y Subvenciones:** Posibles subvenciones para startups tecnológicas.

### 3.1 Diagrama de Secuenciación del Proyecto
Para crear el diagrama de secuenciación, considera las fases del proyecto y sus interdependencias.

### 3.2 Secuenciación de Actividades
Las actividades se desarrollarán en un periodo de 3 meses, comenzando con el análisis y diseño, seguido de la implementación y pruebas.

### 3.3 Asignación de Recursos y Tiempos
- **Recursos Humanos:**
  - 2 desarrolladores frontend.
  - 2 desarrolladores backend.
  - 1 diseñador UX/UI.
- **Tiempos:**
  - Análisis: 2 semanas.
  - Diseño: 3 semanas.
  - Implementación: 6 semanas.
  - Pruebas: 3 semanas.

### 3.4 Procedimientos en las Actividades
